# 隐私政策

**最后更新：** 2026-07-22

## 概述

白泽（BaiZe）是一个以本地为先的 AI 灵魂伙伴应用。我们重视你的隐私，并在开源内测阶段把边界写清楚。

本政策说明：我们如何处理数据、哪些数据只留在本机、以及你使用自备云端模型时会发生什么。

## 数据收集

### 我们默认不收集的数据

**白泽默认不主动收集用户数据用于运营分析或上传到官方服务器。**

当前公开 beta（`v0.9.0-launch`）主路径是：

- 应用安装在你的 Android 设备
- 人格、聊天与设置优先保存在本地
- 云端对话使用**你自己提供的 API Key** 直连你配置的模型服务

### 你主动提供 / 本地产生的数据

- **API 配置**：Base URL、模型名、API Key 等，仅保存在你的设备本地（敏感项加密存储）
- **聊天内容**：与 AI 伙伴的对话内容，默认保存在本地数据库
- **灵魂 / 人格文件**：预置、导入或编辑后的人格与记忆相关文件，默认保存在本地
- **设置信息**：应用偏好、语音与对话相关设置等

## 数据存储

### 存储位置

在当前 beta 主路径下，核心数据优先位于你的 Android 设备本地：

- **聊天记录**：本地 SQLite 数据库
- **灵魂 / 人格文件**：应用内部存储
- **设置信息**：SharedPreferences 等本地配置

### 数据安全

- **加密存储**：敏感配置（如 API Key）使用设备侧加密方案保存
- **本地优先**：聊天与人格默认不经过白泽官方托管账号体系
- **备份保护**：`android:allowBackup="false"`，降低通过 ADB 备份直接导出应用数据的风险

> 说明：没有任何本地应用能在 root / 物理拿到设备等极端情况下提供绝对保密。请妥善保管手机与备份。

## 数据使用

### 我们如何使用这些数据

白泽客户端使用这些数据，是为了在你的设备上提供：

- AI 对话
- 人格切换 / 导入
- 记忆与设置的本地保存

**本 beta 不提供官方云账号同步，也不把聊天内容上传到白泽官方服务器用于训练。**

### 数据共享

**我们不会把你的聊天记录、人格文件或 API Key 卖给第三方。**

但请注意：

- 若你配置了云端模型，消息内容会发送到**你指定的模型服务商**
- 该过程受对应服务商的隐私政策与条款约束
- 白泽无法控制第三方模型服务如何处理请求内容

## 第三方服务

### 云端 API（可选，BYOK）

如果你选择使用 OpenAI 兼容云端接口：

1. 你需要自行提供 API Key
2. 请求会发往你填写的 Base URL 对应服务
3. 白泽不会替你保管官方托管额度账号
4. 剪贴板导出配置默认**不包含** API Key；导入模板后仍需你单独粘贴真实 Key

### 本地模型 / 其他能力

源码中可能保留本地模型、集群、通信等路径，但在 `v0.9.0-launch` 中**未作为产品化主推能力**。若你自行启用相关实验能力，请以实际行为与后续文档为准。

### 支付 / 账号 / 商城

当前公开 beta **默认不开放**：

- 官方注册登录
- 充值
- 提现
- 付费商城

即使源码中存在相关路径，也不等于已开启。

## 权限说明

### 应用权限

| 权限 | 用途 | 是否必要 |
|------|------|----------|
| `INTERNET` | 云端 API 调用、可选资源下载 | 可选（云端对话需要） |
| `ACCESS_NETWORK_STATE` | 检测网络状态 | 是 |
| `RECORD_AUDIO` | 语音输入 | 可选 |
| `READ_EXTERNAL_STORAGE` | 兼容旧版存储读取 | 否（仅低版本 Android） |
| `WRITE_EXTERNAL_STORAGE` | 兼容旧版存储写入 | 否（仅低版本 Android） |

### 权限管理

- `RECORD_AUDIO` 需运行时授权
- 不申请与主路径无关的 `CAMERA` / `LOCATION` / `CONTACTS` 等权限

## 数据删除

你可以随时清理本地数据：

1. 在应用设置中清除聊天记录（若提供该入口）
2. 删除或替换人格 / 灵魂文件
3. 卸载应用以删除应用私有数据

卸载后，应用私有目录中的数据会被系统删除；白泽不保留你的云端官方用户档案（因为本 beta 不提供该体系）。

## 儿童隐私

白泽不面向 13 岁以下儿童。我们不会故意收集 13 岁以下儿童的个人信息。

## 隐私政策更新

我们可能更新本政策。更新后会同步到仓库，并修改“最后更新”日期。重大边界变化会尽量在 README / Release 中同步说明。
